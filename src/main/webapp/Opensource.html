03 — Contribute

Send us
a pull request.

Every ByteLog repository follows the same contribution workflow. We review PRs within 48 hours, and all contributors get credited in the companion article.

1

Fork the repository and git clone your fork locally.

2

Create a branch: git checkout -b fix/your-fix-name

3

Make your changes and run mvn test — all tests must pass.

4

Open a PR with a clear description of what changed and why.

bash
$ git clone https://github.com/bytelog/bytelog-auth
Cloning into 'bytelog-auth'...
✓ done.
 
$ cd bytelog-auth && mvn test
[INFO] Running PasswordUtilTest
[INFO] Running ValidationUtilTest
[INFO] Tests run: 13, Failures: 0
 
$ git checkout -b fix/bcrypt-cost-validation
Switched to a new branch 'fix/bcrypt-cost-validation'
 
$ git push origin fix/bcrypt-cost-validation
